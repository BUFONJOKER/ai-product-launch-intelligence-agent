Task: Modular Structure – Backend Architecture

This page outlines the structural organization of the FastAPI backend. By implementing a Modular Design, we decouple the API routing logic from the agent orchestration and core utilities. This ensures the codebase remains maintainable as we add more specialist nodes or complex tools.

Overview

A flat folder structure leads to "Spaghetti Code" in multi-agent systems. This task establishes a clean hierarchy where each layer of the application has a single responsibility.

  • app/api: Handles the HTTP request/response cycle.
  • app/agents: Houses the LangGraph logic and node definitions.
  • app/core: Contains shared configurations, database connections, and security.

Description

The directory structure is designed to support the uv environment and local development on WSL2. It separates the "Business Logic" (Agents) from the "Infrastructure" (FastAPI/Database).

Folder Hierarchy

Plaintext

app/
├── api/ # API Layer
│ ├── v1/ # Versioned endpoints
│ │ ├── chat.py # Thread & message endpoints
│ │ └── research.py # Research trigger endpoints
│ └── dependencies.py # Auth & DB session injectors
├── agents/ # Agentic Layer
│ ├── graph.py # LangGraph StateGraph definition
│ ├── state.py # TypedDict state & Pydantic models
│ ├── nodes/ # Specialist node logic (Sentiment, Metrics, etc.)
│ └── tools/ # Custom tools (Firecrawl, RAG)
├── core/ # Infrastructure Layer
│ ├── config.py # Pydantic-settings (.env management)
│ ├── database.py # Supabase/Postgres connection pooling
│ └── security.py # API key validation
└── main.py # Entry point

Steps to Complete

  1. Directory Creation:
    • Use mkdir -p to generate the nested folder structure.
    • Initialize each directory with an __init__.py file to ensure they are treated as Python packages.
  1. Configuration (core/):
    • Use pydantic-settings to create a Settings class that loads your Supabase and OpenAI keys.
  1. State Definition (agents/):
    • Define the shared State dictionary and the Pydantic models for structured agent outputs.
  1. Entry Point (main.py):
    • Initialize the FastAPI app and include the routers from app/api/v1.
    • Set up CORS middleware to allow your Next.js frontend to communicate with the backend.
  1. Dependency Injection:
    • Create a utility in dependencies.py that provides the LangGraph compiled graph to the API routes.

Acceptance Criteria

  • Clean Imports: You must be able to import modules using absolute paths (e.g., from app.core.config import settings).
  • Scalability: Adding a new agent node should only require adding a file in app/agents/nodes/ without touching the API code.
  • Separation of Concerns: The main.py file should remain under 50 lines, serving only as a "glue" for the sub-modules.
  • Validation: Running uv run uvicorn app.main:app should show no import errors or circular dependency issues.
