Task: FE-01: Next.js Init

This page outlines the foundational setup for the project's frontend. We are building a modern, type-safe interface that will serve as the control center for our LangGraph agents, utilizing a robust stack for speed, styling, and component consistency.

Overview

Next.js Initialization sets the stage for the entire user experience. By integrating TypeScript, we ensure data flowing from our agents to the UI is strictly typed, reducing runtime errors. Tailwind CSS provides the utility-first styling framework, and shadcn/ui gives us a collection of accessible, high-quality components that we can fully customize.

Description

The goal is to create a performant, scalable repository structure. We will utilize the Next.js App Router for advanced features like server components and streaming, which are essential for displaying real-time agent "thinking" and streaming responses from LangGraph.

Steps to Complete

  1. Project Scaffolding:
    • Create a new Next.js project using the latest stable version.
    • Enable TypeScript, ESLint, and Tailwind CSS during the initialization prompt.
    • Use the src/ directory and App Router layout.
  1. Package Manager Configuration:
    • Initialize the project environment using the uv package manager (via uvx or uv run) to manage JavaScript dependencies if utilizing a polyglot environment, or ensure standard pnpm/npm lockfiles are managed efficiently.
  1. shadcn/ui Integration:
    • Initialize the shadcn/ui CLI.
    • Configure the components.json with preferred styles (e.g., "New York" style) and base colors (e.g., "Slate").
    • Add essential base components: Button, Card, Input, and ScrollArea.
  1. Theme & Global Styling:
    • Set up Dark Mode support using next-themes.
    • Configure the Tailwind theme in tailwind.config.ts to match the brand's aesthetic.
  1. Environment Variable Setup:
    • Create .env.local to store the Supabase URL, Anon Key, and the API endpoint for the LangGraph backend.

Acceptance Criteria

  • Type Safety: The project compiles without any TypeScript errors.
  • UI Foundation: Running the local development server displays a clean landing page utilizing shadcn/ui components.
  • Styling Consistency: Tailwind CSS is functional, and the global CSS correctly imports the shadcn/ui variables.
  • Component Library: The components/ui folder is populated with initialized shadcn/ui primitives.
  • Build Readiness: The project passes a next build check to ensure the initial setup is deployment-ready.
