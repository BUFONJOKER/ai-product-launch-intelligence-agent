Task: Prompt Optimization

This page outlines the strategy for refining the system messages across all specialist nodes. The goal is to maximize the reasoning capabilities of the agents while minimizing token consumption and inference latency, ensuring the LangGraph workflow remains both cost-effective and highly responsive.

Overview

In a multi-agent system, bulky or redundant prompts can lead to "token bloat," which increases costs and slows down the time-to-first-token. Prompt Optimization focuses on "Lossless Compression"—reducing the word count of instructions without losing the behavioral constraints or persona nuances required for high-quality analysis.

Description

Fine-tuning prompts involves moving away from conversational filler and toward structured, high-density instructions. By using techniques like XML tagging, Delimiters, and Negative Constraints, we guide the LLM to focus only on the relevant parts of the graph state, preventing it from re-processing unnecessary context.

Steps to Complete

  1. Instruction Distillation:
    • Audit current prompts to remove "politeness" or repetitive phrases (e.g., changing "Please carefully look at the data and then provide..." to "Analyze data and provide...").
    • Use concise, imperative verbs to define actions.
  1. Structural Optimization:
    • Use XML-style tags (e.g., <context>, <constraints>, <output_format>) to help the LLM parse the prompt structure faster.
    • Implement Variable Injection—ensure only the necessary slices of the State are passed to the prompt rather than the entire message history.
  1. Few-Shot Example Pruning:
    • Replace long, multi-turn examples with 1-2 highly dense "Golden Examples" that demonstrate complex edge cases.
  1. Schema Alignment:
    • Align the system prompt instructions exactly with the Pydantic model field names to reduce the reasoning effort required for structured output generation.
  1. Chain-of-Thought (CoT) Control:
    • For simple tasks (like the Router), disable internal reasoning to save tokens.
    • For complex tasks (like the Collator), use "Hidden Thought" prompts that allow the model to reason before outputting the final JSON.

Acceptance Criteria

  • Token Reduction: Target a 15–25% reduction in total prompt tokens per node without a drop in output accuracy.
  • Latency Improvement: Measurable decrease in "Time to Complete Node" during performance testing.
  • Instruction Adherence: The agent must still follow all core constraints (e.g., "Use UV instead of Pip") despite the shortened instructions.
  • Zero Ambiguity: The optimized prompts must pass a "Stress Test" where the LLM is presented with edge-case data and still follows the intended logic.
  • Format Stability: The agent must continue to output valid JSON/Markdown as defined by the system schemas.
