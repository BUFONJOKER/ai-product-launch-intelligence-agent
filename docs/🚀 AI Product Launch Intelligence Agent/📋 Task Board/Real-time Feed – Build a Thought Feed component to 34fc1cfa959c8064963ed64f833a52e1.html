Task: FE-05: Real-time "Thought Feed" Component

This page outlines the development of the Thought Feed, a critical transparency layer in the UI. Instead of a static loading spinner, this component provides a live, streaming log of the agent's internal reasoning, tool calls, and state transitions, giving the user visibility into the "black box" of the LangGraph workflow.

Overview

The Thought Feed (also known as a "Trace" or "Activity Log") bridges the gap between a user's request and the final report. It streams updates from the server—such as "Sentiment Guru is analyzing Reddit chatter" or "Launch Analyst is cross-referencing pricing"—to keep the user engaged during long-running research tasks.

Description

This component utilizes Server-Sent Events (SSE) or Streaming Responses from Next.js. It renders a chronological list of activities, using distinct icons and colors to represent different nodes in the graph. The feed is designed to be "glanceable," allowing users to understand the agent's progress at a distance.

Steps to Complete

  1. Streaming Endpoint Connection:
    • Configure a React hook (e.g., useStream) to listen to the LangGraph execution endpoint.
    • Parse the incoming stream for node names, tool calls, and metadata.
  1. Feed Architecture:
    • Create a vertically scrolling list using the shadcn/ui ScrollArea.
    • Implement an "Auto-scroll to bottom" behavior that can be toggled off if the user manually scrolls up.
  1. Visual Status Mapping:
    • Node Activity: Display the name of the active specialist with a pulsating "Processing" icon.
    • Tool Execution: Show specific tool icons (e.g., a search icon for Firecrawl) when external data is being fetched.
    • Completion: Transition the status to a checkmark once a node finishes its update.
  1. "Thought" Snippets:
    • If the agent provides internal reasoning (Chain of Thought), display a truncated "preview" of its logic that can be expanded with a click.
  1. State Persistence:
    • Ensure the thought feed is stored in the Supabase thread_id history so users can review the agent's "path to conclusion" even after the research is finished.

Acceptance Criteria

  • Real-time Latency: Updates must appear in the UI within 200ms of the agent emitting a state change.
  • Informative Status: The user should never be left wondering "what is it doing?" during a 60-second crawl.
  • Visual Hierarchy: Critical errors must be highlighted in red, tool calls in blue, and successful completions in green.
  • Performance: The feed must handle hundreds of log entries without causing UI lag or memory leaks.
  • Minimalist Design: The feed should use a "Terminal-lite" aesthetic—monospaced fonts for logs but polished shadcn components for the container.
